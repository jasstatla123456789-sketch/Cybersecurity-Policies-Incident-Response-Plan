# References

## Overview

This project was developed by following internationally recognized cybersecurity frameworks, standards, government publications, and security best practices. These references provide guidance for developing cybersecurity policies, conducting risk assessments, creating incident response procedures, and implementing business continuity and disaster recovery plans.

---

# Cybersecurity Standards

## NIST Cybersecurity Framework (CSF)

**Publisher:** National Institute of Standards and Technology (NIST)

The NIST Cybersecurity Framework provides best practices for managing cybersecurity risks through five core functions:

- Identify
- Protect
- Detect
- Respond
- Recover

Website:
https://www.nist.gov/cyberframework

---

## NIST Special Publication 800-61 Revision 2

**Title:** Computer Security Incident Handling Guide

This publication provides recommendations for developing and managing an Incident Response Program.

Website:
https://csrc.nist.gov

---

## NIST Special Publication 800-53

**Title:** Security and Privacy Controls for Information Systems

Provides a comprehensive catalog of security controls for protecting organizational systems and data.

Website:
https://csrc.nist.gov

---

# OWASP

## OWASP Top 10

The OWASP Top 10 identifies the most critical security risks affecting web applications.

Common risks include:

- Broken Access Control
- Cryptographic Failures
- Injection
- Insecure Design
- Security Misconfiguration
- Vulnerable Components
- Authentication Failures

Website:
https://owasp.org
