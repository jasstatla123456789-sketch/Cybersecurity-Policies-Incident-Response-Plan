# Purpose

The purpose of this Password Policy is to establish requirements for creating, managing, and protecting passwords used to access company systems, applications, networks, and cloud services. Strong passwords are one of the most effective defenses against unauthorized access and cyberattacks.

---

# Scope

This policy applies to:

- Desktop computers
- Laptops
- Mobile devices
- Email accounts
- Cloud services
- VPN access
- Business applications
- Network devices
- Administrator accounts

---

# Password Requirements

All passwords must meet the following minimum requirements:

| Requirement | Standard |
|-------------|----------|
| Minimum Length | 12 Characters |
| Uppercase Letters | Required |
| Lowercase Letters | Required |
| Numbers | Required |
| Special Characters | Required |
| Dictionary Words | Not Allowed Alone |
| Personal Information | Not Allowed |

---

# Strong Password Example

```
Cyber@2026Secure!
```

---

# Weak Password Examples

The following passwords must never be used:

```
password
12345678
admin
company123
welcome
qwerty
abc123
```

---

# Password Creation Guidelines

Users should:

- Create unique passwords for every account.
- Use passphrases whenever possible.
- Avoid predictable patterns.
- Avoid using names, birthdays, or phone numbers.
- Never reuse passwords from personal accounts.

---

# Password Expiration

Passwords must be changed:

| Account Type | Change Interval |
|--------------|-----------------|
| Standard User | Every 90 Days |
| Administrator | Every 60 Days |
| Service Account | Every 180 Days |
| Temporary Account | Immediately After Use |

Passwords must also be changed immediately if compromise is suspected.

---

# Password History

To prevent password reuse:

- The previous **10 passwords** cannot be reused.
- Passwords must be significantly different from previous passwords.

---

# Multi-Factor Authentication (MFA)

MFA is mandatory for:

- Email accounts
- VPN access
- Cloud applications
- Administrator accounts
- Financial systems
- HR systems

Approved MFA methods include:

- Authenticator applications
- Hardware security keys
- Biometric authentication
- One-time verification codes

SMS-based MFA should only be used if stronger methods are unavailable.

---

# Password Storage

Passwords must:

- Never be written on paper.
- Never be stored in plain text.
- Never be shared through email or messaging applications.
- Be stored only in approved password managers.

---

# Password Managers

Employees are encouraged to use approved password managers to:

- Generate strong passwords.
- Store credentials securely.
- Prevent password reuse.

Examples include enterprise-approved password management solutions.

---

# Account Lockout Policy

Accounts will automatically lock after:

- 5 consecutive failed login attempts.

Lockout duration:

- 15 minutes

Repeated failed attempts may require IT assistance to unlock the account.

---

# Password Reset

Password resets require:

- Identity verification.
- Approval from the IT Administrator (if applicable).
- Immediate creation of a new strong password.

Temporary passwords must be changed during the first login.

---

# Administrator Accounts

Administrator accounts must:

- Use unique passwords.
- Never be shared.
- Use MFA.
- Be monitored regularly.
- Be used only for administrative tasks.

---

# Service Accounts

Service accounts should:

- Have unique credentials.
- Be managed by IT.
- Use strong passwords.
- Be reviewed periodically.
- Have access limited to required services.

---

# Shared Accounts

Shared accounts are strongly discouraged.

If unavoidable:

- Approval from management is required.
- Password changes must be documented.
- Access must be limited.
- Activity should be logged.

---

# Password Security Best Practices

Employees should:

- Lock computers when unattended.
- Log out after completing work.
- Report suspicious login attempts.
- Avoid entering passwords on unknown websites.
- Verify website addresses before logging in.

---

# Prohibited Practices

The following actions are prohibited:

- Sharing passwords.
- Reusing company passwords on personal accounts.
- Saving passwords in web browsers without approval.
- Sending passwords through email.
- Using default passwords.
- Writing passwords on sticky notes.

---

# Responsibilities

## Employees

Employees must:

- Create strong passwords.
- Keep passwords confidential.
- Report suspected compromise immediately.
- Follow this policy at all times.

---

## IT Administrator

The IT Administrator is responsible for:

- Enforcing password requirements.
- Managing password resets.
- Monitoring authentication logs.
- Implementing MFA.
- Conducting periodic password audits.

---

# Compliance

Failure to comply with this policy may result in:

- Account suspension.
- Security investigation.
- Disciplinary action.
- Loss of system access.
- Legal action where applicable.

---

# Policy Review

This policy will be reviewed:

- Every 12 months.
- After major cybersecurity incidents.
- Following changes to authentication technologies.
- When regulatory requirements change.
